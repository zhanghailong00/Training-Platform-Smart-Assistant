# 试题生成助手 V1.1 计划 — 稳定输出 JSON

> 目标：构建一套「约束 → 校验 → 修复 → 重试 → 兜底」的 JSON 保障流水线

---

## 整体流程

```
LLM 生成
  ↓ ① DeepSeek JSON Mode（模型层保证语法合法）
  ↓ ② 正则提取 + 解包 questions（文本层修复格式）
  ↓ ③ Pydantic 校验（字段名、字段类型、必填项）
  ↓ ④ 业务规则校验（单选题1个答案、多选题至少2个等）
  ↓ ⑤ 校验失败 → 把错误喂给 LLM 重新生成 → 最多 3 次
  ↓ ⑥ 3 次全失败 → 跳过该题型，返回已成功的
最终输出
```

---

## 要做 4 件事

### 1. DeepSeek JSON Mode

**改了哪个文件**：`components/llm_client.py` + `prompts/exam_generation.yaml`

**具体怎么做**：

调用 DeepSeek API 时加一个参数：

```python
response = client.chat.completions.create(
    model=...,
    messages=messages,
    response_format={'type': 'json_object'},  # 这一行
)
```

同时，JSON Mode 要求输出必须是 JSON 对象 `{...}`，不能是数组 `[...]`。所以 Prompt 里的输出格式从：

```json
[ { "type": 0, ... }, { "type": 1, ... } ]
```

改为：

```json
{ "questions": [ { "type": 0, ... }, { "type": 1, ... } ] }
```

Prompt 末尾加一句：`"输出必须为 JSON 对象，所有试题放在 questions 字段中。"`

**这层解决什么问题**：LLM 不会输出 markdown 代码块、不会输出解释文字、不会输出注释。从模型层保证输出语法合法的 JSON。

---

### 2. 文本提取 + 解包 questions

**改了哪个文件**：`utils/json_utils.py`

**具体怎么做**：

在已有的 4 层 JSON 提取策略之后，加一个解包逻辑：

```python
data = extract_json(raw_response)
# 如果 JSON Mode 输出了 {"questions": [...]}，解包成数组
if isinstance(data, dict) and "questions" in data:
    data = data["questions"]
```

原有的 4 层提取策略不变（直接解析 → 匹配代码块 → 匹配最外层数组 → 匹配最外层对象）。

**这层解决什么问题**：把 JSON Mode 的 `{"questions": [...]}` 格式转回数组，下游不需要改任何代码。

---

### 3. Pydantic 校验（已有，纳入流程）

**在哪个文件**：`components/validator.py`

**具体怎么做**：

对提取出来的每条数据，用 Pydantic 模型校验：

```python
from schemas.question import Question

for item in data:
    try:
        question = Question.model_validate(item)
        # 校验通过
    except ValidationError as e:
        # 校验失败，记录错误原因
        errors.append(f"字段校验失败: {e.errors()}")
```

Pydantic 会自动检查：

| 检查项 | 示例 |
|---|---|
| 字段名是否正确 | `content` 写成了 `conten` → 报错 |
| 字段类型是否正确 | `level` 传了字符串 `"0"` 而不是整数 `0` → 报错 |
| 必填字段是否缺失 | 没有 `analysis` 字段 → 报错 |
| 可选字段格式是否正确 | `subAnswer` 传了数组而不是字符串 → 报错 |

校验失败的题会被收集错误原因，不直接丢弃——错误原因会喂给 LLM 做自修复重试。

**这层解决什么问题**：JSON 语法合法不等于字段合法。Pydantic 保证字段名、类型、必填项都正确。

### 4. 业务规则校验（新增）

**新增文件**：`components/business_validator.py`

**具体怎么做**：

```python
def validate_business_rules(questions: list) -> list[str]:
    """校验业务规则，返回错误信息列表"""
    errors = []
    for i, q in enumerate(questions):
        if q.type == 0:  # 单选题
            right = sum(1 for o in q.options if o.isRight)
            if right != 1:
                errors.append(f"单选题 [{i}] 应有 1 个正确答案，实际 {right} 个")
            if len(q.options) != 4:
                errors.append(f"单选题 [{i}] 应有 4 个选项，实际 {len(q.options)} 个")
        elif q.type == 1:  # 多选题
            right = sum(1 for o in q.options if o.isRight)
            if right < 2:
                errors.append(f"多选题 [{i}] 至少应有 2 个正确答案，实际 {right} 个")
        elif q.type == 2:  # 判断题
            contents = {o.content for o in q.options}
            if contents != {"正确", "错误"}:
                errors.append(f"判断题 [{i}] 选项必须为'正确'和'错误'")
        elif q.type == 3:  # 简答题
            if q.options:
                errors.append(f"简答题 [{i}] options 应为空")
            if not q.subAnswer:
                errors.append(f"简答题 [{i}] subAnswer 不能为空")
    return errors
```

**这层解决什么问题**：JSON Mode 和 Pydantic 都管不了业务规则。单选题出了 2 个正确答案，JSON 是合法的，Pydantic 也是通过的，但题目是错的。这一层专门管这个。

### 5. 自修复重试 + 降级返回

**改了哪个文件**：`services/exam_service.py`

**具体怎么做**：

对每种题型，最多尝试 3 次：

```
第 1 次：
  调 LLM（JSON Mode）
  → 提取 + 解包 questions
  → Pydantic 校验
  → 业务规则校验
  → 全部通过 ✓  → 成功，进入下一种题型
  → 有错误     → 进入第 2 次

第 2 次：
  把第 1 次的校验错误拼到 Prompt 末尾：
  "你之前生成的题目有以下问题，请修正后重新输出：
   1. 单选题 [0] 应有 1 个正确答案，实际 2 个
   2. 判断题 [1] 选项必须为'正确'和'错误'"
  → 重新调 LLM
  → 重新校验
  → 全部通过 ✓  → 成功
  → 仍有错误   → 进入第 3 次

第 3 次：同上

3 次全失败 → 跳过该题型，记录失败原因
```

所有题型处理完后：

- **有成功题目** → 返回 `success: true` + 已成功题目 + `error` 字段说明哪些题型失败
- **全部失败** → 返回 `success: false` + 错误信息

**这层解决什么问题**：
- 自修复：不是盲重试，是把错误告诉 LLM 让它有针对性地修正，成功率远高于简单重试
- 降级：一个题型失败不阻塞整个流程，用户体验更好

## 总结：做完后的 JSON 保障流水线

| 步骤 | 谁做 | 解决什么问题 |
|---|---|---|
| ① JSON Mode | `llm_client.py` | 保证语法合法，杜绝 markdown/解释文本 |
| ② 提取 + 解包 | `json_utils.py` | 从 LLM 响应中提取 JSON，解包 questions |
| ③ Pydantic 校验 | `validator.py` | 保证字段名、类型、必填项正确 |
| ④ 业务规则校验 | `business_validator.py` | 保证单选题1个答案、判断题2个选项等 |
| ⑤ 自修复重试 | `exam_service.py` | 校验失败时把错误喂给 LLM，最多修正 3 次 |
| ⑥ 降级返回 | `exam_service.py` | 部分成功不阻塞，全部失败才报错 |
